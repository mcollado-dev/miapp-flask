Funciones Disponibles

Esta sección describe las principales funcionalidades implementadas en la aplicación Flask, así como su propósito dentro del flujo general del sistema y del pipeline de despliegue automatizado.

Gestión de Usuarios

La aplicación implementa un modelo de datos Usuario gestionado mediante SQLAlchemy y almacenado en una base de datos MariaDB externa. Cada registro contiene un nombre, un correo electrónico y un rol asociado. Esta estructura permite una gestión de usuarios flexible y persistente en un entorno real, eliminando la dependencia de bases de datos locales de prueba.

Generación de Estadísticas

La aplicación incorpora una sección de estadísticas que emplea Matplotlib para generar gráficos dinámicos en formato base64. Estos gráficos muestran la distribución de roles entre los usuarios y se renderizan directamente en la interfaz HTML sin necesidad de archivos físicos. El proceso está totalmente automatizado y optimizado para entornos web.

Exportación y Análisis de Datos

El sistema está preparado para futuras extensiones que permitirán exportar información de usuarios y métricas en formatos como CSV o PDF. También admite el desarrollo de módulos analíticos más avanzados que procesen los datos directamente desde la base de datos MariaDB para obtener métricas de actividad o segmentaciones personalizadas.

Configuración y Seguridad

La aplicación utiliza variables de entorno y configuraciones específicas según el entorno (development, testing o production). La conexión con la base de datos se gestiona de forma segura a través de una credencial protegida en Jenkins, evitando almacenar contraseñas en el código fuente. Esto mejora la seguridad y permite una integración profesional con los pipelines de CI/CD.

Despliegue Automatizado

El proceso de despliegue está completamente automatizado mediante Jenkins y Docker. El pipeline realiza las siguientes operaciones:

  • Descarga el código fuente y ejecuta los tests unitarios.
  • Analiza la calidad del código con SonarQube.
  • Construye la imagen Docker de la aplicación Flask.
  • Verifica la conexión con la base de datos MariaDB mediante una credencial segura.
  • Despliega la aplicación en el servidor remoto 192.168.56.106 y valida su disponibilidad.

Gracias a esta integración, cada actualización del repositorio puede ser probada, analizada y desplegada automáticamente, garantizando un flujo de entrega continua y un entorno de producción controlado.
